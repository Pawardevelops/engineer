## Problem In Simple Words

Return all unique quadruplets whose sum equals the target.

The important skill is to identify what the array is really asking you to control: indexes, a window, a running state, sorted order, or an in-place transformation.

## Why This Pattern Fits

After sorting, fix two values and solve the remaining pair with two pointers. Duplicate skipping at each level prevents repeated answers.

This matters because the optimized solution is not just faster by accident. It is faster because the pattern gives us a rule for safely ignoring work that cannot improve the answer.

## Brute Force Thinking

Check every group of four values and deduplicate afterward. That is O(n^4).

Brute force is still worth understanding first because it defines the answer clearly. Once you know exactly what brute force checks, the optimized pattern is easier to trust.

## Optimized Intuition

The optimized idea is to keep only the state that can still affect the final answer.

For this problem:

- The current state tells us what candidates are still possible.
- The update rule removes candidates that are now impossible or already handled.
- The answer is updated only when the state represents a valid candidate.

## Step-By-Step Algorithm

1. Sort nums.
2. Fix the first index and skip duplicates.
3. Fix the second index and skip duplicates.
4. Run two pointers for the remaining target.
5. After a hit, move both pointers and skip duplicates.

## Python Solution

```python
from typing import List


class Solution:
    def fourSum(self, nums: List[int], target: int) -> List[List[int]]:
        nums.sort()
        result = []
        n = len(nums)

        for i in range(n - 3):
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            for j in range(i + 1, n - 2):
                if j > i + 1 and nums[j] == nums[j - 1]:
                    continue

                left, right = j + 1, n - 1
                while left < right:
                    total = nums[i] + nums[j] + nums[left] + nums[right]

                    if total == target:
                        result.append([nums[i], nums[j], nums[left], nums[right]])
                        left += 1
                        right -= 1

                        while left < right and nums[left] == nums[left - 1]:
                            left += 1
                        while left < right and nums[right] == nums[right + 1]:
                            right -= 1
                    elif total < target:
                        left += 1
                    else:
                        right -= 1

        return result
```

## Complexity Analysis

Time O(n^3), space O(1) extra besides output.

## Common Mistakes

- Skipping duplicates only at the pointer level but not the fixed indexes.
- Using the same index twice.
- Returning duplicate quadruplets because sorted duplicate logic is incomplete.

## What You Should Learn

You learn the reusable k-sum idea: sort, fix values, reduce to a two-pointer pair search.
